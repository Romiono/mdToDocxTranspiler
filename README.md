# MD → DOCX Транспилятор (ГОСТ 7.32-2017)

Конвертирует Markdown-файлы в DOCX, соответствующий требованиям ГОСТ 7.32-2017 и методическим указаниям КубГУ 2019. Подходит для ВКР, курсовых, дипломных и магистерских работ.

## Возможности

- Автоматическое форматирование по ГОСТ: поля, шрифт Times New Roman 14 пт, межстрочный интервал 1,5
- Структурные заголовки (ВВЕДЕНИЕ, ЗАКЛЮЧЕНИЕ и т.д.) — прописные, по центру
- Нумерованные разделы до 4 уровней вложенности
- Автонумерация таблиц, рисунков, формул (сквозная или по разделам)
- **Формулы в нативном формате Word (OMML)** — редактируются в Word через встроенный редактор уравнений
- Автоматическое оглавление (поле TOC, обновляется в Word по `Ctrl+A` → `F9`)
- Нумерация страниц в нижнем колонтитуле
- Приложения с буквенной нумерацией (А, Б, В… по ГОСТ)

## Установка

**Требования:** Python 3.8+

### Вариант 1 — через виртуальное окружение

```bash
python3 -m venv .venv
source .venv/bin/activate      # Windows: .venv\Scripts\activate
pip install -e .
```

После этого доступна команда `md2docx` внутри venv:

```bash
md2docx input.md [output.docx]
```

### Вариант 2 — без установки пакета

```bash
pip install -r requirements.txt
python3 transpiler.py input.md [output.docx]
```

### Вариант 3 — через `python -m`

```bash
pip install -r requirements.txt
PYTHONPATH=src python3 -m md_to_docx_gost input.md [output.docx]
```

## Использование

```bash
md2docx input.md [output.docx]
# или без установки:
python3 transpiler.py input.md [output.docx]
```

Если `output.docx` не указан, файл сохраняется рядом с входным с тем же именем.

**Примеры:**
```bash
md2docx examples/test6_sample_vkr.md
md2docx my_thesis.md my_thesis.docx
md2docx tests/test5_statistics.md tests/test5_statistics.docx
```

## Структура MD-файла

### YAML-шапка (обязательна)

```yaml
---
type: vkr          # vkr | kursovaya | diplomnaya | magisterskaya | nir
title: Название работы
author: Фамилия Имя Отчество
year: 2024
---
```

### Порядок структурных элементов

```markdown
# СОДЕРЖАНИЕ
# ВВЕДЕНИЕ
# 1 Первый раздел
## 1.1 Подраздел
### 1.1.1 Пункт
#### 1.1.1.1 Подпункт
# 2 Второй раздел
# ЗАКЛЮЧЕНИЕ
# СПИСОК ИСПОЛЬЗОВАННЫХ ИСТОЧНИКОВ
# ПРИЛОЖЕНИЕ А
> Название приложения
```

## Синтаксис элементов

### Заголовки

```markdown
# ВВЕДЕНИЕ                    ← структурный (ALL CAPS, по центру)
# 1 Название раздела          ← раздел (жирный, с абзацного отступа)
## 1.1 Название подраздела    ← подраздел
### 1.1.1 Название пункта     ← пункт
```

> Точка после номера НЕ ставится: `# 1 Название` ✓ — `# 1. Название` ✗

### Текст и форматирование

| Синтаксис | Результат |
|-----------|-----------|
| `*текст*` | курсив (термины, определения) |
| `` `код` `` | моноширинный (имена переменных, код) |
| `[14]` | затекстовая ссылка |
| `[14, с. 25]` | ссылка с указанием страницы |
| `$формула$` | inline-математика в формате OMML |

### Списки

**С тире (основной):**
```markdown
- первый элемент;
- второй элемент;
- третий элемент.
```
Дефис `-` автоматически заменяется на тире `–`. После каждого элемента, кроме последнего, — точка с запятой; после последнего — точка.

**С буквенной нумерацией:**
```markdown
а) первый подпункт;
б) второй подпункт;
в) третий подпункт.
```

**Вложенный с цифрами:**
```markdown
а) первый пункт:
1) первый подпункт;
2) второй подпункт;
б) второй пункт.
```

### Таблицы

Подпись ставится **выше** таблицы в формате HTML-комментария или именованной строкой:

```markdown
<!-- Таблица 2.1 – Название таблицы -->
| Заголовок 1 | Заголовок 2 | Заголовок 3 |
|-------------|-------------|-------------|
| Данные 1    | Данные 2    | –           |
```

```markdown
Таблица 3.2 – Динамика показателей
| Показатель | 2022 | 2023 |
|------------|------|------|
| ВВП, млрд  | 135  | 148  |
```

Нумерация: сквозная (`Таблица 1`) или по разделам (`Таблица 2.1`).

### Рисунки

```markdown
![Описание рисунка](путь/к/файлу.png)
```

Транспилятор автоматически добавляет подпись `Рисунок N – Описание` ниже рисунка. В приложениях: `Рисунок А.1 – Описание`.

### Формулы

**Отдельная строка:**
```markdown
$$\bar{x} = \frac{1}{n} \sum_{i=1}^{n} x_i$$
```

**С явным номером:**
```markdown
$$\rho = \frac{m}{V}$$ (1)
```

**Расшифровка символов (блок «где»):**
```markdown
$$K = \frac{P}{Q} \cdot 100$$

где $K$ – показатель эффективности, %;
- $P$ – результирующий показатель;
- $Q$ – базовый показатель.
```

Слово «где» — с маленькой буквы, без двоеточия. После каждого символа, кроме последнего, — точка с запятой; после последнего — точка.

Формулы конвертируются в **нативный Word Math (OMML)** и редактируются в Word через встроенный редактор уравнений.

> Поддерживаемые LaTeX-команды: все стандартные. Команды `\displaystyle`, `\textstyle`, `\bigl`, `\bigr`, `\left`, `\right` удаляются автоматически.

### Приложения

```markdown
# ПРИЛОЖЕНИЕ А

> Название первого приложения

Текст приложения...
```

Допустимые буквы приложений (по ГОСТ): А Б В Г Д Е Ж И К Л М Н П Р С Т У Ф Х Ц Ш Щ Э Ю Я  
(буквы Ё З Й О Ч Ъ Ы Ь — не используются)

После исчерпания кириллицы: A B C D E F G H J K L M N P Q R S T U V W X Y Z (без I и O).

### Специальные элементы

**Принудительный разрыв страницы:**
```markdown
---
```
или
```markdown
<!-- pagebreak -->
```

**Примечания:**
```markdown
*Примечание* — Текст примечания.
```

### Библиографический список

```markdown
# СПИСОК ИСПОЛЬЗОВАННЫХ ИСТОЧНИКОВ

1. Иванов И. О. Название книги / И. О. Иванов. – М.: Издательство, 2020. – 320 с.

2. Петров П. П. Название статьи / П. П. Петров // Журнал. – 2021. – № 3. – С. 15–22.

3. Название сайта [Электронный ресурс]. – URL: https://example.com (дата обращения: 01.01.2024).
```

Ссылки в тексте: `[1]`, `[1, с. 45]`, `[2, 3]`

## Чек-лист перед конвертацией

- [ ] YAML-шапка присутствует
- [ ] Структурные заголовки прописными буквами: `# ВВЕДЕНИЕ` ✓, `# Введение` ✗
- [ ] Нет точки после номера раздела: `# 1 Название` ✓, `# 1. Название` ✗
- [ ] Подписи таблиц — **выше** таблицы в `<!-- ... -->` или именованной строкой
- [ ] Путь к рисункам — относительный от расположения MD-файла
- [ ] Блок «где» — без двоеточия после слова «где»
- [ ] Список источников — нумерованный список
- [ ] Приложения обозначены буквами: `# ПРИЛОЖЕНИЕ А` ✓, `# ПРИЛОЖЕНИЕ 1` ✗

## Оглавление в Word

После открытия DOCX нажмите `Ctrl+A`, затем `F9` — Word обновит поле СОДЕРЖАНИЕ и подставит реальные номера страниц.
